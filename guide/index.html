Overview

Kubernetes Introduction

In this guide you will get a quick introduction to containerization and Kubernetes. It will help you set Kubernetes up on your local machine and deploy a basic workload to it.

Installation

We will use Docker Desktop which is free for smaller companies and costs a small fee per developer for large companies.

Containers

A container is an “isolated process”, it’s easiest to just think of it as a VM.

Pods

In Kubernetes, a “Pod”1 is the smallest deployable unit possible. This is kind of an abstract description.

Deployment

Creating a pod is cool, but it’s not that different from just running the Docker container by hand.

Service

In Kubernetes the term “Service”1 is used for loadbalancers inside the cluster. In this section, we will explore how to communicate with multiple pods using these loadbalancers so you can communicate with any of pods through a single interface.
