Containers

A container is an “isolated process”, it’s easiest to just think of it as a VM.1 The benefits of containers is that you have a self-contained run-time environment that describes how to run your application. For example, let’s say I had written a Java server application, and I hand it off to you and say “run it”. What are you supposed to do with it? First, you need to figure out which version of Java it needs, then you need to know if it requires a configuration file, is it supposed to run inside TomCat or as a stand-alone server?
